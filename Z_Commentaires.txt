Pas de dossier node_modules
C'est logique dans ton cas car tu utilises Docker.
Pourquoi ? Regarde tes Dockerfile : tu fais RUN npm install (ou npm ci) à l'intérieur de l'image Docker. Les dépendances sont donc installées dans le conteneur, pas sur ton ordinateur (ton host).
Le problème : Ton éditeur (VS Code) ne trouve pas les bibliothèques pour l'autocomplétion.
La solution : Pour avoir l'autocomplétion et enlever les erreurs rouges dans ton code, tu devrais ouvrir un terminal dans tes dossiers frontend/ 
et backend/ (sur ta machine, pas dans Docker) et taper : npm install
Cela créera les dossiers node_modules localement pour ton éditeur.

L'autocomplétion est une fonctionnalité des éditeurs comme VS Code qui propose automatiquement des suggestions de code (noms de fonctions, variables, modules) pendant que vous tapez, pour accélérer l'écriture et réduire les erreurs.
Option ou nécessaire ?
C'est une option, pas une nécessité pour que votre code fonctionne ou que Docker tourne correctement.
​Aantages pratiques
Gain de temps : Évite de retaper ou mémoriser les APIs des bibliothèques comme Fastify ou React.
Moins d'erreurs : Signale les problèmes (types, imports manquants) avant l'exécution.
Découverte : Montre les méthodes disponibles sur un objet (ex: req.body. suggère les propriétés).
Dans votre cas Docker
Sans node_modules local, VS Code ne "voit" pas les dépendances installées dans le conteneur, d'où les erreurs rouges.
npm install local (frontend/backend) crée juste ce dossier pour l'éditeur (~500 Mo, ignoré par Docker via .dockerignore).
Alternative : Utilisez le Dev Container de VS Code pour un environnement Dockerisé avec autocomplétion intégrée.
Pour un dev full-stack comme vous, c'est fortement recommandé pour la productivité
